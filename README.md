## ✨ 功能特色

- 📧 **邮箱验证码验证**：新成员入群后自动发送验证码至其QQ邮箱，群内直接发送验证码即可完成验证，无需 @机器人
- ⏱️ **超时自动踢出**：超过设定时间未验证自动移出群聊，支持超时前倒计时提醒
- 🔄 **答错自动换码**：验证错误后自动生成新验证码并重发邮件，超过重试次数直接踢出
- 🛡️ **管理员跳过验证**：群主、群管理员或机器人管理员可使用 `/skip @某人` 命令直接跳过指定用户的验证
- 🏢 **分群白名单**：支持指定特定群号启用验证，未在名单内的群不触发验证
- 🎨 **完全可自定义**：所有消息提示语均可自由配置，支持丰富的模板变量
- 🔒 **多群隔离**：同一用户在不同群的验证状态互不干扰，退群自动清理任务

## 📋 验证流程
新成员入群
    ↓
机器人发送验证码邮件 → {QQ号}@qq.com
    ↓
机器人在群内 @新成员 提醒查收
    ↓
新成员在群内直接发送验证码
    ↓
  ✅ 正确 → 验证通过
  ❌ 错误 → 换码重发邮件（超过上限踢出）
  ⏰ 超时 → 踢出群聊
  🛡️ 管理员 /skip @某人 → 直接跳过
  
  ## ⚙️ 配置说明

所有配置均可在 AstrBot WebUI 插件配置页面中直接修改，无需编辑代码。

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| SMTP 服务器地址 | string | `smtp.qq.com` | QQ邮箱默认即可 |
| SMTP 端口 | int | `465` | SSL 端口 |
| 发件邮箱地址 | string | - | 你的QQ邮箱地址 |
| 邮箱授权码 | string | - | QQ邮箱设置中获取，非QQ密码 |
| 发件人显示名称 | string | `QQ群验证系统` | 收件人看到的发件人名 |
| 启用验证的群号列表 | list | `[]` | 留空则对所有群生效 |
| 验证超时时间 | int | `300` | 单位：秒，默认 5 分钟 |
| 超时警告提前时间 | int | `60` | 设为 0 禁用警告 |
| 踢出前延迟 | int | `5` | 发送超时通知后多久踢人 |
| 最大答错次数 | int | `3` | 设为 0 不限制 |
| 验证码长度 | int | `6` | 验证码位数 |

## 📝 QQ邮箱授权码获取

1. 登录 [QQ邮箱网页版](https://mail.qq.com)
2. 进入 **设置 → 账户**
3. 找到 **POP3/SMTP 服务**，点击开启
4. 按提示发送短信验证，获取 **16位授权码**
5. 将授权码填入插件配置的「邮箱授权码」

## 💬 使用命令

| 命令 | 权限 | 说明 |
|------|------|------|
| `/skip @某人` | 群主 / 群管理员 / 机器人管理员 | 跳过指定用户的验证 |
| `/跳过验证 @某人` | 同上 | 同上（中文别名） |

## ❓ 常见问题

**Q: 收不到验证码邮件？**
A: 请检查邮箱授权码是否正确，以及QQ邮箱 POP3/SMTP 服务是否已开启。部分邮箱可能会将验证邮件归入垃圾箱，请注意查收。

**Q: 机器人没有踢人权限？**
A: 请确保机器人帐号拥有群管理员或群主权限。

**Q: 如何只对特定群启用验证？**
A: 在配置的「启用验证的群号列表」中填入群号即可。留空则对所有群生效。

**Q: 支持哪些消息平台？**
A: 目前仅支持 aiocqhttp（OneBot v11 协议），如 NapCat、Lagrange 等。
